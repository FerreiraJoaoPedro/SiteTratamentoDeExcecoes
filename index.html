Tratamento de Exceções

Tuquis

O que é uma exceção?

O tema desse site é tratamento de exceções, mas antes de entender como tratá-las nós precisamos entender o que elas são e o que significam.
O termo exceção é usado para designar um evento que ocorre durante a execução de um programa que desvia o fluxo normal de instruções, em outras palavras, uma exceção é uma condição provocada por uma situação excepcional que requer uma ação específica imediata.
Simplificando mais ainda, exceções são situações que o programa não está preparado para revolver, e que por esse motivo são encaradas muitas vezes como erros por interromperem o funcionamento do programa de alguma forma
Agora que já sabemos que exceções são indesejadas, podemos pensar em uma forma de como combatê-las para deixar o programa mais fluido e estável.

O Try Catch

Não é por que exceções são eventos inesperados que nós não podemos prevenir a aparição deles.
Pensando nisso foi criado o "Try Catch".
O Try Catch é uma forma de tratamento de exceções, ou seja, ele é um bloco de código que nós colocamos em uma parte que possa culminar em um evento problemático para que ele gere uma saída mais conveniente e menos destrutiva para a aplicação.
Já sabemos o que são exceções e que existe uma forma de preveni-las: o Try Catch. Sabendo disso, vamos conhecer a fundo esse comando e como utiliza-lo com sua total eficiência.

Como utilizá-lo?

O Try Catch é basicamente dividido em duas partes: o try e o catch, com uma função cada uma.
O "try", como o próprio nome é a parte do comando que vai tentar executar o código que pode gerar a exceção.
O "catch" é a parte do comando que irá executar a medida para resolver a exceção, é dentro dele que escrevemos o que o programa deverá fazer no caso de uma exceção ocorrer.
Vejamos agora um exemplo de como esse comando pode ser utilizado.
